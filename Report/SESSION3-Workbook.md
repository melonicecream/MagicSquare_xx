## 1) 주제 한 문장 (Mom Test 기반, 솔루션 최소화)

> **4×4 부분 마방진(빈 칸 2, 합 34)을 채운 뒤, 행·열만으로 “거의 끝”이라 판단해 대각선을 늦게 확인하다 틀림을 늦게 알고 약 20분을 되돌리는 일을, 10선(4행·4열·2대각선) 판정으로 줄인다.**

*(앱·Solver·ECB “프로그램 만들기”는 주제에 넣지 않음.)*

---

## 2) R-G-I-O

| | 내용 |
|---|------|
| **Role** | 4×4 부분 마방진을 **손으로** (또는 과제용 **코드로**) 빈칸 2개를 채우고 행·열·대각선 합을 확인하는 **학습자** |
| **Goal** | 빈칸을 채운 상태에서 **10선 합 34**를 끝까지 확인하고, “대각선 제외하고는 다 맞다”는 **잘못된 완료 판정** 없이 pass/fail을 확신한다 |
| **Input** | 4×4 정수 격자 (`0` = 빈 칸 **정확히 2개**, 나머지 1~16 일부 기입), 목표 합 **34** |
| **Output** | **통과/실패** + 실패 시 **깨진 선 ID** (`row:*`, `col:*`, `diag:main`, `diag:anti`) — “행·열만 맞음” 수준의 모호한 결과는 Goal 미달 |

---

## 3) 성공 기준 3개 (Mom Test 증거와 연결)

| # | 성공 기준 | 연결 증거 |
|---|-----------|-----------|
| **S1** | 행·열은 통과인데 **대각선만** 깨진 격자를 넣으면 `diag:*` 실패를 **한 번의 판정**으로 반환한다 | ② “대각선을 **제외하곤** 다 맞았다고 생각” → ③ “**대각선**이었어” |
| **S2** | 판정은 **행·열·대각선 10선 전부**를 검사한다. 대각선을 빼면 통과로 오판하지 않는다 | ① “**대각선 하나를 빼먹어**서 20분 날렸다” |
| **S3** | 동일 격자에 대해 **재실행·재검증**해도 같은 pass/fail·같은 실패 선 목록이 나온다 (수동 합 계산 반복 대체) | ③ “**숫자 합을 내서 검증**을 하다보니 알았어” — 수동 검증 비용·재현 |

---

## 4) 표면 문제 — 이번 프로젝트에서 **하지 않을 것**

| 하지 않음 | 이유 (Mom Test) |
|-----------|-----------------|
| “4×4 마방진 **ECB 패턴으로 프로그램** 만든다” | 솔루션·과제 형식이지 학습자 **완료 착각·20분 재작업** 증거와 직접 연결되지 않음 |
| **`SquareValidator` / `Solver` 클래스 구현**을 목표로 삼기 | 클래스 이름·아키텍처는 표면; 이번엔 **판정 행동**만 |
| **마방진 앱·UI**로 빈칸 채우기 돕기 | “대신 풀기/보기 좋게” ≠ “대각선 늦게 알아차림” |
| **TDD부터** 설득 (“테스트 먼저 쓰면…”) | 인터뷰에 없는 구현 방법론 |
| 빈칸 **자동 완성 Solver** | 문제는 “채우기”가 아니라 **검증·판정·완료 기준** |

**이번에 하는 것:** 10선 합 34 **판정** + 실패 선 식별 + Test Loop — 채우기 알고리즘·UI·ECB 전체는 후속.

## 8계층 — **이번 세션에서 만드는 것만**

| 계층 | 이번 세션 | 내용 |
|------|-----------|------|
| **Rule** | ✅ | 4×4, `0` 정확히 2개, 1~16(중복 없음), **10선** 합 = 34 |
| **Command** | ✅ | `validate(grid)` → `{ ok, violations[] }` — 깨진 행·열·대각선 ID 반환 |
| **(Skill)** | △ 선택 | `listViolations(grid)` — 실패 선만 요약 (Solver 아님) |
| **Test Loop** | ✅ | S1~S3 fixture + Red→Green: **행·열만 검사** → 대각선 누락 시 S1/S2 실패 → **10선 검사**로 Green |
| *(Boundary, Entity 저장, Solver, Policy 등)* | ❌ | 후속 세션 |

### Test Loop 개념

- **Red:** 행·열만 검사 → 대각선만 틀린 격자에서 **통과 오판** (증거 ②·③ 재현)
- **Green:** 10선 검사 → `diag:main` 또는 `diag:anti` 반환 (S1)
- **Refactor:** 동일 입력 2회 → 동일 `violations` (S3)
