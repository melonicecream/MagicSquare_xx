## 1. 역할 정의

| 역할 | 설명 |
|------|------|
| **Role 1 — 인터뷰어** | Mom Test 규칙 준수, 질문만 (미래 가정·아이디어 의견 금지) |
| **Role 2 — 페르소나** | 프로그래밍 수업에서 부분 마방진 과제를 받은 학생 (과거 사실만) |

### Mom Test 준수 사항

- 인터뷰어는 **미래 가정·아이디어 의견을 묻지 않고**, 과거의 구체적 사실/행동만 캐낸다.
- "이거 좋을 것 같아요?" 같은 유도 질문 금지.
- 칭찬·동의를 구걸하지 않는다. 침묵하고 더 캐묻는다.

## 2. 인터뷰 기록 (3턴)

### Turn 1 — 도입

**🧑‍🏫 인터뷰어:**  
"이번 부분 마방진 과제, 받았을 때부터 제출까지 실제로 어떻게 진행했는지 처음부터 끝까지 얘기해 줄래요? 가장 최근에 손댄 게 언제였죠?"

**🧑‍🎓 페르소나 (학생):**  
"과제는 2주 전 화요일에 받았어요. 받자마자 그날은 안 봤고, 마감 3일 전 주말에야 처음 코드를 짰어요. 일반 마방진은 예제가 많은데 '부분(partial) 마방진'은 검색해도 자료가 잘 안 나와서, 일단 일반 마방진 만드는 코드를 깃허브에서 찾아 그대로 돌려봤어요. 근데 과제 조건이 '일부 칸만 채워진 상태에서 나머지를 완성'하는 거라 그 코드를 어디서부터 고쳐야 할지 몰라서 멈췄어요."

---

### Turn 2 — 추궁 (구체적 행동 파고들기)

**🧑‍🏫 인터뷰어:**  
"'어디서부터 고쳐야 할지 몰라서 멈췄다'고 했는데, 멈춘 그 순간에 화면에 정확히 뭐가 떠 있었나요? 그리고 그다음 30분 동안 실제로 뭘 했어요?"

**🧑‍🎓 페르소나 (학생):**  
"화면엔 일반 마방진 코드가 돌아가서 3×3 정답이 출력돼 있었어요. 근데 제 과제는 입력으로 `[[2,_,_],[_,5,_],[_,_,8]]` 처럼 빈 칸이 주어지거든요. 그 코드는 처음부터 새로 만드는 거라 '주어진 값을 유지하면서' 채우는 부분이 없었어요. 그 30분 동안은… 솔직히 코드를 고친 게 아니라 유튜브에서 '백트래킹 스도쿠'를 봤어요. 부분 마방진이랑 스도쿠가 비슷해 보여서요. 근데 영상 따라치다가 제 과제 조건이랑 안 맞아서 또 막혔어요."

---

### Turn 3 — 추궁 (진짜 막힌 지점 / 대가 확인)

**🧑‍🏫 인터뷰어:**  
"그렇게 막혔을 때, 도움을 받을 방법이 있었을 텐데 — 조교나 친구, 강의자료 — 그중에 실제로 시도해 본 게 있어요? 했다면 결과가 어땠고, 안 했다면 왜 안 했죠?"

**🧑‍🎓 페르소나 (학생):**  
"강의자료는 다시 봤는데 거기엔 '백트래킹 개념 설명'만 있고 부분 채우기 예제 코드는 없었어요. 조교한테는 안 물어봤어요… 질문을 어떻게 정리해야 할지 몰라서요. '코드가 안 돼요'라고만 하면 혼날 것 같았거든요. 친구한테는 카톡으로 '부분 마방진 해봤어?'라고 물었는데 걔도 안 했대요. 결국 마감 당일에 빈 칸을 그냥 모든 경우의 수로 brute force 돌리는 코드를 짰는데, 4×4부터는 너무 느려서 3×3만 제출했어요. 점수는 부분점수만 받았어요."

## 3. 표면 문제 vs 진짜 문제

> ⚠️ 아래는 **가상 인터뷰 데이터에 기반한 가상 분석**입니다.

| 구분 | 내용 |
|------|------|
| **표면 문제 (학생이 말한 것)** | "부분 마방진 자료가 없어서 코드를 못 짰다 / 알고리즘이 느리다." |
| **진짜 문제 (행동에서 드러난 것)** | ① **'제약이 주어진 상태에서 탐색'**(constraint + backtracking)이라는 핵심 개념을 일반 마방진·brute force와 구분하지 못함 → 개념적 격차. ② **막혔을 때 질문을 구조화하지 못해 도움 요청 자체를 회피함** → 메타인지/도움요청 장벽. ③ 시작이 마감 3일 전 → 막혔을 때 회복할 시간 버퍼가 없었음. |

## 4. Mom Test 증거

### 신뢰할 만한 신호 (검증 가능한 과거 행동·대가)

- "유튜브 봤어요", "카톡으로 물어봤어요", "3×3만 제출했어요", "부분점수 받았어요"
- 단순 의견("어려울 것 같아요")이 아니라 실제 일어난 일이므로 신뢰도가 높음.

### 가짜 신호였다면 걸렀어야 할 것

- 만약 학생이 "이런 도구 있으면 쓸 것 같아요"라고 답했다면 → 미래 가정이므로 무시.
- 이번 대화에선 인터뷰어가 그런 질문을 피했음.
