# MagicSquare_xx — Mom Test 인터뷰 종료 · 결과 추출

> 인터뷰 Q&A가 끝난 뒤 붙여넣어 실행

---

## 프롬프트 (복사용)

```
Mom Test 인터뷰 종료.
지금까지 답변:
[워크북 전체 붙여넣기]

다음만 출력:
1. 표면 문제 (잘못된 정의) — "~~프로그램 만든다" 류
2. 진짜 문제 (한 문장) — 불편·비용·판정·재현 중심, 솔루션 이름 없음
3. Mom Test 증거 3줄 (내 답에서 인용)
```

---

## 출력 기준

| 항목 | 작성 규칙 |
|------|-----------|
| **표면 문제** | 솔루션·기능·아키텍처가 섞인 **잘못된 정의** ("~~프로그램 만든다", "~~앱", "Validator/Solver 구현") |
| **진짜 문제** | **한 문장**, 불편·비용·판정·재현 중심, TDD/PyQt/Cursor 등 솔루션名 없음 |
| **증거 3줄** | 인터뷰 **답변에서 직접 인용** (질문 문장 X) |

---

```markdown
## Mom Test 워크북 — MagicSquare_xx

- (시점 / 행동 / 시간 / 실수)

### 표면 문제 (금지 — 솔루션 섞인 문장):
- (솔루션형으로 말했을 법한 것 — 검증 안 된 가정)

### 진짜 문제 (한 문장):
(종료 프롬프트 실행 후 확정)
```

## 채점 프롬프트 (선택)

```
아래 Mom Test 결과를 채점해줘. 수정하지 마.
[워크북 전체]

체크:
- [ ] 미래 가정("~하면 좋겠다") 없음
- [ ] 과거 행동·시간·실수 구체성 있음
- [ ] 진짜 문제에 솔루션名(TDD/PyQt/Cursor) 없음
- [ ] 표면 문제와 진짜 문제가 분리됨
- [ ] MagicSquare 도메인(4×4, 10선, 34, 빈칸 2개) 반영

점수 /10 + 고칠 질문 1개만 제안
```
