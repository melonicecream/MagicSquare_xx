# Mom Test 역할 분리 시뮬레이션 (MagicSquare_xx)

> 재사용 프롬프트 — Cursor 등에 그대로 붙여넣어 실행

---

## 프롬프트 (복사용)

```
역할 분리 Mom Test 시뮬레이션 (MagicSquare_xx).

Role 1 — 인터뷰어: Mom Test 규칙 준수, 질문만
Role 2 — 페르소나: "프로그래밍 수업에서 부분 마방진 과제를 받은 학생"
  (가상이지만 plausible한 과거 사실만)

3턴 진행:
- Turn 1: 인터뷰어 질문 → 페르소나 답
- Turn 2: 추궁 → 답
- Turn 3: 추궁 → 답

마지막: 표면 문제 vs 진짜 문제 (가상 인터뷰임을 명시)

⚠️ 내 실제 경험과 섞지 말 것. 시뮬레이션 라벨 필수.
```

---

## Role 1 — 인터뷰어 규칙 (Mom Test)

### 해야 할 것

- 과거의 **구체적 사실·행동**만 묻는다.
- "그때 화면에 뭐가 보였나요?", "그다음 30분 동안 뭘 했나요?"처럼 **추궁**한다.
- 침묵을 두려워하지 않고, 답이 모호하면 더 캐묻는다.

### 하지 말아야 할 것

- "이 기능 있으면 쓸 것 같아요?" — **미래 가정** 질문 금지
- "이런 앱/도구 만들면 좋을까요?" — **솔루션 제안** 질문 금지
- "좋은 아이디어네요", "그거 필요하겠네요" — **칭찬·동의 구걸** 금지
- 페르소나에게 **어떻게 고치면 좋을지** 묻지 않는다

---

## Role 2 — 페르소나 규칙

| 항목 | 내용 |
|------|------|
| 역할 | 프로그래밍 수업에서 부분(partial) 마방진 과제를 받은 학생 |
| 답변 범위 | **과거에 실제로 일어났을 법한** 구체적 사실만 (가상이지만 plausible) |
| 금지 | 사용자(프롬프트 작성자)의 실제 경험과 섞기 |
| 톤 | 솔직하지만, "어려울 것 같아요" 같은 의견보다 **했던 행동** 위주 |

### plausible한 디테일 예시 (참고용, 그대로 복붙하지 말 것)

- 일반 마방진 코드를 찾아 돌려봤지만, "주어진 칸 유지" 조건과 안 맞음
- 유튜브·깃허브·강의자료·친구·조교 등 **실제 시도한 도움 경로**
- 마감 압박, brute force, 부분 제출, 부분점수 등 **구체적 대가**

---

## 출력 형식

1. 문서 상단에 **시뮬레이션 라벨** (가상 인터뷰임을 명시)
2. Turn 1 ~ Turn 3: 인터뷰어 질문 → 페르소나 답 (각 턴 구분)
3. 마지막 섹션: **표면 문제 vs 진짜 문제** 표 + Mom Test 증거 요약
